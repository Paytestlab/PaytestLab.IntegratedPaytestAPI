# Release Notes (Version 1.0.2)

## Overview
Improved transaction handling and webhook management with enhanced data structures and new endpoints.

## Key Changes

### StartTransactionRequest Enhancements
- **Added `terminalId`:** Identifies the terminal where the transaction is initiated.
- **Added `transactionInformation`:**
  - New object containing details such as `transactionType`, `amount`, `amountTip`, `amountOther`, and `referenceNumber`.
- **Added optional headers** `X-TestRun-Id` and `X-TestCase-Id` for passing test metadata without polluting the request body.
- **Added optional `callbackTimeoutSeconds`:** Indicates how long the caller will wait for the transaction completion callback before timing out.

### TransactionCompletionNotification Updates
- **Receipt Conversion:**
  - Changed `receiptMerchant`, `receiptCardHolder`, and `receiptCashRegister` from string fields to use a new `Receipt` object.
- **Added `hostResponse`:**
  - New object providing details with properties: `responseCode`, `responseMessage`, and `transactionDateTime`.
- **Property Renaming:**
  - Renamed `status` to `transactionStatus` for clarity and consistency.

### New Objects Introduced
- **TransactionInformation Object:**
  - `transactionType`
  - `amount`
  - `amountTip`
  - `amountOther`
  - `referenceNumber`
- **Receipt Object:**
  - `content`
- **HostResponse Object:**
  - `responseCode`
  - `responseMessage`
  - `transactionDateTime`

### Additional Endpoint Updates
- **GET /status:**
  - Added a required query parameter `terminalId` to uniquely identify the terminal.
  
  
- **New Webhook Endpoints:**
  - **POST /webhooks/register:** Registers a webhook for terminal events (e.g., terminal state changes).
  - **POST /webhooks/unregister:** Unregisters an existing webhook.
