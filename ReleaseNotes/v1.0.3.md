# Release Notes (Version 1.0.3)

## Overview
Introduced session management capabilities, improved transaction validation, and enhanced API error handling to provide better transaction correlation and reliability during test execution.

## Key Changes

### Session Management

#### New Session Endpoints
Added support for managing test sessions through two new endpoints:

- **POST /session/initiate**
  - Creates a new session for a specified terminal.
  - Returns a `sessionId` used to correlate subsequent transactions.

- **POST /session/complete**
  - Completes an active session.
  - Returns a summary containing:
    - Session result
    - Number of executed transactions
    - Transaction statuses
    - Authorized amounts

### Transaction Updates

#### StartTransactionRequest
- **Added required `sessionId`**
  - Transactions must now be associated with an active session.

#### AbortTransactionRequest
- **Added required `sessionId`**
  - Abort requests now validate both the transaction and session context.

### Session Reporting

#### New Response Objects

**CompleteSessionResponse**
- `sessionId`
- `result`
- `transactionCount`
- `transactions`
- `errorMessage`

**SessionTransactionSummary**
- `transactionId`
- `transactionStatus`
- `authorizedAmount`

### Error Handling Improvements

#### Additional Error Responses
Added support for `404 Not Found` responses where applicable, including:

- Session not found
- Transaction not found
- Terminal not found

#### Standardized Error Responses
Expanded use of the `ErrorResponse` schema across session and transaction endpoints for improved troubleshooting and diagnostics.

### Validation Improvements

#### Amount Object
Added required fields:

- `value`
- `currency`

This ensures all monetary values are submitted with complete currency information.

### Documentation Updates

- Expanded endpoint documentation.
- Added session lifecycle documentation.
- Added transaction-to-session correlation guidance.
- Improved response and error handling documentation.
